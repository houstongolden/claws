Two modes. One mandate: ship.

Claws is built around a simple constraint: if it’s code, you get a working artifact. If it’s research, you get sources. Everything else is theater.

Vibe coding

Bold UI that compiles into a file you can open.

Fast iteration, strong aesthetics, and a single deliverable: a page that actually renders. No "concept" screenshots. No placeholders pretending to be a product.

`fs.write` artifacts responsive layouts inline CSS
Plan → Execute

Constraint-driven output that stays readable.

You get a short plan first, then the work. It keeps momentum high and prevents the classic agent failure mode: improvising forever and shipping nothing.

2–4 step plan tool-first execution clean structure
Research & computer use

Evidence-aware answers, not confident poetry.

Claws can search, fetch, extract, and screenshot—then summarize with linked sources. If a page is JS-heavy, it switches tactics.

web search doc fetching page extraction
Local-first

Your files, your paths, your control.

Writes to your repo structure, not a mystery cloud. The interface is immediate: open the artifact, tweak it, commit it. Repeat.

predictable output no lock-in repo-native

Durable findings and follow-ups, on demand.

When you want it, Claws persists key details and spins up tasks—so the project keeps moving even after the chat scrolls away.

Minimal adjectives. Maximum specificity.
