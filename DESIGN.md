# Claws Design System

Generated by /design-consultation on 2026-04-08
Status: DRAFT v1

## Product Context

Claws is a multi-agent harness. Users are solo founders and developers who want to orchestrate AI agents autonomously. Local-first, Vercel-native stack. Current dashboard works but looks generic. This design system leans into "indie control room" — mission control aesthetic for AI agent orchestration.

## Aesthetic Direction

**"Indie control room"** — the feeling of a hacker's mission control for AI agents.

- Dense with live data
- Confident with red accent
- Mono-heavy for terminal-adjacent vibe
- No decoration beyond function
- Every pixel earns its place (Rams: "as little design as possible")

The hero experience is NOT "chat with an AI." It's "orchestrate a team of AIs." The UI must reflect orchestration, not conversation.

## Typography

**Sans:** Geist Sans (keep current)
- Headings and body
- Use weights 400, 500, 600, 700, 800
- Tracking: -0.011em for body, -0.02em for headings

**Mono:** Berkeley Mono (paid, preferred) or JetBrains Mono (free fallback)
- All code, CLI commands, event logs, agent ids
- Current Geist Mono is fine for v1 but upgrade to Berkeley Mono if budget allows

**Display:** Geist Sans at heavy weights (700/800)
- No separate display font. Discipline over variety.

**Sizes:**
- xs: 11px (labels, badges)
- sm: 13px (body small, captions)
- base: 14px (body)
- lg: 16px (emphasis)
- xl: 18px (subheadings)
- 2xl: 22px (section headings)
- 3xl: 28px (page titles)
- 4xl: 40px (hero — rare)

## Motion

- **Framer Motion** for state transitions (already planned for week 3)
- **Hover:** 200ms ease-out
- **Node spawn/despawn:** 400ms spring (stiffness 300, damping 30)
- **Working pulse:** 2s cycle, 0.5 → 1.0 opacity, ease-in-out
- **NO page transitions, scroll animations, parallax**

## Design Decisions Log

### 2026-04-08: Initial design system
- **Decision:** Red accent (#ff3344) as primary brand color
  - **Why:** Every competitor uses cool colors (Linear purple, Cursor blue, Lovable purple). Red is empty. Claws gets a face.
  - **Tradeoff:** Red is aggressive, easy to overdo. Mitigation: discipline — use only for active state, CTAs, errors.

- **Decision:** Mission control density over Linear-style airiness
  - **Why:** Power users want data visible. Claws orchestrates agents, not conversations.
  - **Tradeoff:** Intimidates beginners. Mitigation: collapsible panels, "simple mode" toggle.

- **Decision:** Keep Geist Sans + Geist Mono for v1
  - **Why:** Already working, Vercel-native, zero migration cost.
  - **Future:** Upgrade to Berkeley Mono if budget allows ($75/license).

- **Decision:** No light theme for v1
  - **Why:** Every developer tool is dark-first. Light theme is a maintenance burden. Revisit after beta.

- **Decision:** Agent tree as the hero component
  - **Why:** Differentiates from "chat + sidebar" layouts. Shows the actual product value.
  - **Risk:** New component, no existing library. Must build from scratch with Framer Motion.

## Design System — Components

### Buttons
- Primary: red background, white text, 14px, 8px/16px padding, 8px radius
- Secondary: transparent, 1px border (surface-3), white text, same sizing
- Ghost: transparent, no border, muted text, hover to surface-2
- Icon: 32x32, rounded, hover to surface-2

### Cards
- Background: surface-1
- Border: 1px solid surface-3
- Radius: 12px
- Padding: 20px
- Shadow: 0 1px 2px rgba(0,0,0,0.4)

### Inputs
- Background: surface-1
- Border: 1px solid surface-3
- Radius: 8px
- Padding: 10px 14px
- Focus: 1px solid accent + 2px accent-dim ring

### Status Dots (agent tree nodes)
- 8x8 circle
- idle: muted
- working: info (pulsing)
- blocked: warning
- done: success
- error: accent

### Typography Components
- Heading 1: 3xl, weight 700, tracking -0.02em
- Heading 2: 2xl, weight 600, tracking -0.02em
- Heading 3: xl, weight 600, tracking -0.015em
- Body: base, weight 400, leading 1.55
- Caption: sm, weight 400, muted
- Code: mono, sm, weight 400, surface-2 background, 2px/6px padding

## Anti-patterns (DO NOT)

- ❌ No purple gradients (every AI startup)
- ❌ No 3-column icon grids (Lovable/v0 cliché)
- ❌ No centered hero text with decorative blobs
- ❌ No glassmorphism beyond the top bar
- ❌ No emoji in UI chrome (emoji in user content is fine)
- ❌ No light theme (v1 scope)
- ❌ No Inter, Roboto, Arial (use Geist Sans)
- ❌ No parallax, no scroll animations, no auto-playing demos
