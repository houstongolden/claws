## Agent-expanded version (my reading of intent)

This is a fundamental repositioning of Claws. The 2026-04-07 design doc and CEO plan assumed Claws was another agent OS competing with Claude Code, ultraworkers/claw-code, and OpenClaw. That framing was honest but generic. The new framing is sharper and solves a real pain point:

**There is no established front-end framework for OpenClaw.** OpenClaw is a backend agent OS with a CLI. Anyone who wants a GUI has to roll their own. Most users never ship a GUI because building one from scratch is expensive. That's the pain point Claws solves.

### Two pillars, same turborepo

**Pillar 1: `@claws/sdk` + Claws Studio + Template Marketplace** (the framework)

This is the main product. Inspired by and extracted from the hubify work, it includes:

- `@claws/sdk` — React hooks + core WebSocket gateway client for talking to OpenClaw. 17 hooks covering skills, tools, presence, config, sessions, dashboard config, pairing, smart polling, ACP sessions. Already exists in `hubify/packages/claws-sdk/src/` (30 files).
- **Gateway Protocol v3** — typed RPC over WebSocket (JSON-RPC-style) for OpenClaw calls. Defined in `hubify/packages/claws-sdk/src/core/protocol.ts`.
- **Claws Studio** — visual template builder. Components at `hubify/apps/web/components/studio/` (DeployModal, PreviewRenderer, StudioLayout, VibeCoder). Libs at `hubify/apps/web/lib/studio/` (themes, widget-grid, widget-primitives, generation-prompts, template-config, file-validator). API routes for `generate` and `deploy`.
- **AIOS Template system** — every template packages SOUL.md (personality), AGENTS.md (instructions), skills/, dashboard layout, themes, cron jobs. 7+ built-in templates referenced in the extraction doc.
- **Template Gallery & Marketplace** — Next.js routes for browsing, publishing, forking templates. Convex-backed in hubify (`convex/templates.ts`, `templateVersions.ts`). For Claws standalone, adapt or swap the backend.
- **SmartSync versioning** — three-way merge algorithm for keeping deployed workspaces up to date with template upgrades without losing user customization.
- **Fly.io provisioning** — full pipeline for deploying a persistent OpenClaw VPS per user, custom subdomain, TLS cert, nginx reverse proxy, JWT auth, stats server. `hubify/infra/workspace/` + `hubify/apps/web/app/api/workspaces/route.ts` + `hubify/convex/squadCompute.ts`.
- **Docker image architecture** — multi-service boot (nginx + OpenClaw + stats server + ttyd web terminal). `hubify/infra/workspace/Dockerfile` + `boot.sh` + `nginx.conf.template` + `openclaw.json.template`.
- **Workspace runtime** — volumes, subdomain routing, stats server HTTP API with 20+ endpoints, workspace sync (CLI ↔ cloud for SOUL/memory/skills).
- **CLI/MCP integration** — whatever hubify ships for pushing workspaces to cloud and syncing.
- **Mintlify docs** — `hubify/docs/claws/` + `hubify/docs/integrations/claws-sdk.mdx`.

**Pillar 2: Experimental Agent OS for Vercelians** (the existing Claws codebase)

This is the learning/research/experimental artifact. The gateway, dashboard, worker, CLI REPL, multi-agent stubs, PGlite runtime, tools. Kept as-is for now.

Long-term role: once the front-end framework is mature, the experimental OS becomes one of the target backends (alongside OpenClaw). Someone could use `@claws/sdk` to build a dashboard that talks to either OpenClaw OR the Claws experimental gateway.

- `@claws/sdk` published on npm as a standalone package
- `claws.so` landing page leads with "front-end framework for OpenClaw UIs"
- At least one real OpenClaw user uses `@claws/sdk` to build a custom dashboard (not a demo, a real workspace)
- Template marketplace has 5+ templates, at least 2 contributed by non-founder users
- Fly.io provisioning works end-to-end: user clicks "deploy," gets `their-name.claws.so` subdomain with live OpenClaw workspace
- Claws Studio can generate + deploy a template in under 2 minutes

**Pillar 2 (experimental OS) success:**
- Stays buildable and documented as a reference implementation
- Eventually consumes `@claws/sdk` itself (dogfooding) — Claws dashboard uses its own SDK
- Becomes a "second backend target" that framework consumers can optionally run locally instead of OpenClaw
